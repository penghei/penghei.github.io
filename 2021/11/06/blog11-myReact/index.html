react知识点汇总和使用小总结(一)

react 本身

react 是一个更加接近原生 js 的框架.相比于 vue,他其中的很多渲染操作都是通过纯 js 来完成,比如条件渲染,循环渲染,实质上都只是 jsx 的条件输出,循环输出而已.因此这样的 react 看起来似乎更加简单,也更加需要对原生 js 的掌握能力.
react 是通过虚拟 dom 的 diffing 算法进行页面更新的,通过 jsx 以及本身自带的一些生命周期,利用 state 和 props 实现单向数据流来更新 ui 的一个框架.这个定义或许不太准确,但是确实有 react 的几大特征.

  • props 是入口,state 是出口
  • 页面更新的方式:state 的改变或父组件 state 的改变
  • 生命周期的实现,可以在组件从挂载到销毁之间完成很多事情
  • 创建了位于 jsx 和原生 dom 的中间层,用于监听事件和执行 diffing 算法:即通过最小代价来对比 jsx 和 state 等的变化,从而改变 ui
  • …..

渲染

React 元素是不可变对象。一旦被创建，你就无法更改它的子元素或者属性。一个元素就像电影的单帧：它代表了某个特定时刻的 UI。更新 UI 唯一的方式是创建一个全新的元素，并将其传入 ReactDOM.render()。
React DOM 会将元素和它的子元素与它们之前的状态进行比较，并只会进行必要的更新来使 DOM 达到预期的状态。
render()是改变界面的唯一方法,只有在 state 改变和父组件变化时才会调用 render.函数式组件没有明写 render,但是当需要重新渲染的时候实际上也就是调用整个函数可以调用的地方,然后进行 return 的 diffing 比对,进行渲染

组件

组件首字母必须大写!!!
按照官方文档的例子,组件调用时是这样的

  1. 我们调用 ReactDOM.render() 函数，并传入 <Welcome name="Sara" /> 作为参数。
  2. React 调用 Welcome 组件，并将 {name: 'Sara'} 作为 props 传入。
  3. Welcome 组件将 <h1>Hello, Sara</h1> 元素作为返回值。
  4. React DOM 将 DOM 高效地更新为 <h1>Hello, Sara</h1>。
    也就是说,组件的调用实际上是给 render 函数传了一个参数,然后父组件把 props 作为参数再传入;紧接着组件收到 props 执行,返回一个或一组 html 元素给父组件,最后展示在调用该组件的父组件上.在脚手架中所有组件都挂载到 App 上,然后 App 单独挂载到 html 文件的 root 根元素上,由此完成了整个组件树的创建
    props 是只读的,不能更改

props

如果父组件传来的 props 变化,那么直接使用 props 的元素也会响应式的更新.换句话说 props 变化意味着父组件一定变化了,也就会连带子组件重新渲染
如果想要监听 props,class 组件可以使用静态方法 getDerivedStateFromProps

在 hooks 可能没有直接的方法,但是可以通过 state 存上一次的 props,对比这次的,不一样就更新

state

state 是组件内部维护的一个状态,通过 setState 可以使组件重新渲染.setState 执行时,会将传入的参数对象与组件当前的状态合并，然后触发调和过程(Reconciliation)。经过调和过程，React 会以相对高效的方式根据新的状态构建 React 元素树并且着手重新渲染整个 UI 界面(diffing 算法)。
如果在短时间内频繁 setState。React 会将 state 的改变压入栈中，在合适的时机，批量更新 state 和视图，达到提高性能的效果。
注意,setState 一般是异步的

  • 异步： 在 React 可以控制的地方，比如在 React 生命周期事件和合成事件中，都会走合并操作，延迟更新的策略。
  • 同步： 在 React 无法控制的地方，比如原生事件，具体就是在 addEventListener 、setTimeout、setInterval 等事件中，就只能同步更新。
    既然是异步的,根据 js 异步任务的特点,同步完成之前是不会执行异步的,所以你就不可以直接把 setState 之后的 state 拿出来使用;但是可以找个变量先接到更改后的并使用.如果希望全局,可以用 useRef 存一下

生命周期

虽然在 hooks 中弱化了生命周期的概念,但是个人感觉要知道的还是得知道

挂载

constructor

  • 初始化组件的 state
  • 给事件处理方法绑定 this

getDerivedStateFromProps

可以看上面 props 的那一块
该函数会在装载时，接收到新的 props 或者调用了 setState 和 forceUpdate 时被调用。如当接收到新的属性想修改 state ，就可以使用。

componentDidMount

需要注意的一点:如果在 componentDidMount 中调用 setState ，就会触发一次额外的渲染，多调用了一次 render 函数，由于它是在浏览器刷新屏幕前执行的，所以用户对此是没有感知的

更新

shouldComponentUpdate

它是在重新渲染组件开始前触发的，默认返回 true，可以比较 this.props 和 nextProps ，this.state 和 nextState 值是否变化，来确认返回 true 或者 false。当返回 false 时，组件的更新过程停止，后续的 render、componentDidUpdate 也不会被调用。

getSnapshotBeforeUpdate(prevProps, prevState)

它使您的组件可以在可能更改之前从 DOM 捕获一些信息（例如滚动位置），在聊天气泡页中用来计算滚动高度。它返回的任何值都将作为参数传递给 componentDidUpdate()
执行时机： 在 render 之后，可以读取但无法使用 DOM 的时候。

componentDidUpdate(prevProps, prevState, snapshot)

该方法有三个参数：

  • prevProps: 更新前的 props
  • prevState: 更新前的 state
  • snapshot: getSnapshotBeforeUpdate()生命周期的返回值

卸载

componentWillUnmount()

在此方法中执行必要的清理操作：

  • 清除 timer，取消网络请求或清除
  • 取消在 componentDidMount() 中创建的订阅等；

不能使用 setState 的生命周期

状态提升

按照官方文档的说法,状态提升就相当于两个组件共享一个状态,比如两个输入框想同时显示一个值,但是每个组件也需要操作一下数据.
这样最好的方法就是把两个组件的state都提到共同父组件的props中,然后更改props就可以同时更改两个state
