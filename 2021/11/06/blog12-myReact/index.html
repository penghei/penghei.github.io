context 用在穿过多个组件的用处,比如组件和孙组件的数据传递,或者两个不相关组件的的传递
一般通过const context = React.createContext()创建,括号填初值
然后在创建的组件中用 provider 包裹

在需要接收的组件使用useContext(),括号是React.createContext()创建的 context 对象

useState

set 的时候 state 是替换而不是合并,一般也是异步的
官方文档中,多个 state 可以合并到一个对象中用 state 存储
注意:使用 push/pop/append 等直接更改数组无法更新 state，应该采用析构方式比如[...state,obj]

useEffect

基本使用就不说了;
useEffect 的精髓在于后面的依赖数组:useEffect 不是一个生命周期的替代,官方文档说它更多是一个副作用的体现,只是可以替代几个生命周期

不用依赖

相当于 componentDidMount 和 componentDidUpdate

依赖为空

相当于 componentDidMount,只在第一次挂载会执行

有依赖

当依赖改变时才会调用.这样理解:比如一个组件需要根据 redux 的值来改变 state,一般就可以这样:

但是这个东西很难说,总之还是要慢慢理解,一般情况下有依赖是最好的:也就是说useEffect中引用的变量尽量都放在依赖中

useLayoutEffect

useLayoutEffect 会在所有的 DOM 变更之后同步调用，主要用于处理 DOM 操作、调整样式、避免页面闪烁等问题.
如果实在分不清，先用 useEffect，一般问题不大；如果页面有异常，再直接替换为 useLayoutEffect 即可。

useCallback

返回一个回调函数。在依赖参数不变的情况下，返回的回调函数是同一个;每当依赖参数发生改变,useCallback就会自动重新返回一个新的 memoized 函数
理解:是一种优化方式,相当于是给一个函数包了一层useCallback.当某个依赖改变时,函数也会随之改变,然后调用时也会调用新的函数

useMemo

返回一个缓存值,在依赖参数不变的的情况返回的是上次第一次计算的值.跟useCallback差不多,可以直接拿这个返回的缓存值用

虚拟dom和diffing

从本质上来说，Virtual Dom是一个JavaScript对象，通过对象的方式来表示DOM结构。通过事件处理机制，将多次DOM修改的结果一次性的更新到页面上(就是前面说的多个state一同更新)，从而有效的减少页面渲染的次数，减少修改DOM的重绘重排次数，提高渲染性能。
在代码渲染到页面之前，react会把代码转换成一个对象（虚拟DOM）。以对象的形式来描述真实dom结构，最终渲染到页面。在每次数据发生变化前，虚拟dom都会缓存一份，变化之时，现在的虚拟dom会与缓存的虚拟dom进行比较。react内部封装了diff算法，通过这个算法来进行比较，渲染时修改改变的变化，原先没有发生改变的通过原先的数据进行渲染。
理解一下就是,react依据代码创建了一个虚拟dom,然后根据前后比较来更新页面,比较的算法是diffing算法,可以最小化消耗

diffing比对策略:

  • 两个dom树对比不同类型元素(同一层级)
  • 对比同类型元素,会对比属性差别
  • 同级子节点,会通过key对比.key主要为了追踪哪个元素被修改/被添加/被删除,从而帮助减少渲染
