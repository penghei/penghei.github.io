react+typescript使用pubsub时的一点小问题

发表于2021-11-22|更新于2021-12-10|工具
|字数总计:350|阅读时长:1分钟|阅读量:

问题

关于 useState()初始值的问题:useState 的参数初始值会在其他 state 导致的页面更新后发生奇怪的更新,也就是说会导致 state 值变为参数所传递的初始值;这个发现来源于某项目的一个 bug:

  • 使用 PubSub 在 useEffect 中订阅,当收到值之后再获取 state 发现为空(也就是初始值)
  • 当把初始值定为另外一个数组的时候,同理也会出现 state 变成初始值的情况

    仔细探究

  • 首先 PubSub 的传输并没有导致页面更新,在 react 的 devtools 中也没有变化,但是获取的时候却确实是初始值.
  • 而当不用 PubSub 而是用其他方式代替,就不会出现这种 bug.所以当前只能确定问题可能跟 PubSub 相关,但是并不能确定具体问题.
  • 经验就是尽量减少 PubSub 在复杂组件和状态中的使用
    自己另写了一个例子尝试了一下,发现问题原因:很大可能是因为在带[]的useEffect中使用导致的,因为去掉[]后就没有这个问题了;可能是如果只是挂载时订阅,组件更新会延迟;换成更新钩子就正常了

    总结

    PubSub使用的useEffect不要带[]了,并且如果还是出现问题,就换成别的方法.
