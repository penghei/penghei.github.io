react状态管理库-Recoil

发表于2022-01-15|更新于2022-01-15|React
|字数总计:664|阅读时长:2分钟|阅读量:

这几天看到了一个很好用的 react 状态管理库，有一点像之前的 hooks 做状态管理的方法，这里写一篇记录一下简单用法，以及未来问题解决

安装配置

安装很简单，这个安装会同时带上 ts 的类型文件，一次安装就不必重新配置 ts 形式，下面也会说到 ts 的写法几乎和 js 没有太大改变；

在 main.js/main.ts 中引入<RecoilRoot>，包在<App/>外面

然后在外部创建文件，文件名任意，写入 recoil 的核心:atom和selector

核心概念

atom

解释主要点：

  • 一个 atom 相当于 redux 的一个 reducer，维持一个状态；当 atom 的状态改变，所有持有 atom 实例的组件状态都会被改变
  • atom 是一个参数为对象的函数，对象的属性常用的有两个：
    • key：表示当前状态名，必须，后续在组件内部的useRecoilState等 hooks 需要作为参数来区分是哪个状态
    • default：默认值，必须，类型为 any
  • 建议把 key 和变量名写成相同的

selector

selector 可以理解为 atom 的一个 getter，可以对获取的 atom 状态进行操作。

  • selector 的 key 是区分 selector 的，不能和任何其他 selector 或 atom 一样；用了 selector 处理的 atom 就不能再用 atom 的 key 引入，需要用处理该 atom 的 selector 的 key 引入
  • 属性 get 就是 atom 的 getter，参数是解构得到的 get 函数，可以用来获取 atom 的返回值；通过在 get 中更改返回可以对 atom 维持的状态获取不同的返回

hooks

在组件中使用三个主要的 hooks 引入

  • useRecoilState：相当于 useState，也是用作代替 useState 的 hook，用法和 useState 完全一样，参数是 atom 的变量名；
  • useRecoilValue：useRecoilState 的简写，返回值只有状态
  • useSetRecoilState：同样的简写，只有操作函数

在typescript中

Recoil 的 ts 配置几乎没有任何障碍，和在 js 中一模一样，useRecoilState 的返回值根据 atom 的 default 类型自动确定，不需要额外的声明文件
